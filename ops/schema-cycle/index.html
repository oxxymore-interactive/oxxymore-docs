Cycle de travail — Notion → HacknPlan → Unity → GitHub → Discord → Décision

Un diagramme visuel, pensé pour être compris en 15 secondes. Unity = atelier. GitHub = preuve. Le reste supporte le flux.

1 outil = 1 fonction
Lecture recommandée : Comprendre→Affecter→Produire →Prouver→Signaler→Décider

Lecture rapide

Notion = compréhension

Règles, process, repères. On lit. On ne “livre” rien ici.

HacknPlan = affectation

Les tâches sont assignées et priorisées. Chaque tâche doit être traçable sur GitHub.

Unity = production

On développe localement. Unity seul n’est pas une preuve.

GitHub = preuve

Commits/PR/Issues. Sans GitHub, le travail est invisible.

Discord = signal

Daily + coordination courte. Toujours avec lien GitHub. Jamais de livrables.

Décision = sans réunion

GO/NO-GO basé sur preuves et éléments vérifiables.

Règle non négociable

OXXYMORE ne juge jamais le temps.
OXXYMORE juge uniquement la preuve.
